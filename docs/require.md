# Flutter 跨平台 Notion 风格 Markdown 所见即所得编辑器 需求文档

## 文档版本：V1.1

## 适用平台：iOS / Android / Web / Windows / macOS

## 核心定位：**纯单视图 WYSIWYG 编辑器**，编辑区即渲染区，无分栏、无源码预览分离，1:1 复刻 Notion 编辑体验

---

## 一、核心定义（强制不可违背）

1. **真正的所见即所得（True WYSIWYG）**
   * **同一个区域**内完成“输入 -> 渲染 -> 修改”，输入 Markdown 语法后**即时自动渲染为富文本样式**，不再显示原始 Markdown 标记符。
   * **绝对禁止**：左右分栏（左边源码、右边预览）的传统模式。
   * **对标基准**：Notion / Typora 的编辑体验。
2. **Block 级编辑**
   * 文档由独立的“块”（Block）组成（段落、标题、列表项、代码块等）。
   * 支持块的拖拽排序、删除、转换类型。

---

## 二、功能需求详情

### 1. 基础编辑体验

* **实时语法渲染**
  * 输入 `# ` + 空格，当前行自动变为 H1 标题样式。
  * 输入 `**文字**`，文字自动变粗，星号消失。
  * 输入 `` `代码` ``，自动渲染为行内代码样式。
  * 输入 `- ` 或 `* `，自动变为无序列表项。
* **光标与选区**
  * 支持在渲染后的富文本中任意位置点击、拖拽选中。
  * 选区跨越不同格式时，保持样式显示正确。
* **快捷键支持**
  * `Cmd/Ctrl + B`：加粗
  * `Cmd/Ctrl + I`：斜体
  * `Cmd/Ctrl + K`：插入链接
  * `Tab` / `Shift + Tab`：列表缩进/取消缩进
  * `Enter`：在当前块下方新建段落块
  * `Backspace`：在空行按 Backspace 删除当前块或合并到上一块
    
    ### 2. Markdown / Notion 特色语法支持
    
    | 功能       | 触发方式                   | 效果描述               |
    |:-------- |:---------------------- |:------------------ |
    | **标题**   | `#` / `##` / `###` ... | H1-H6 字号层级         |
    | **待办事项** | `[ ]` 或 `[x]`          | 可勾选的复选框，勾选后文字自动删除线 |
    | **代码块**  | \`\`\` + 语言名           | 多行代码块，支持语法高亮（可选）   |
    | **引用块**  | `>`                    | 灰色背景/左侧边框的引用样式     |
    | **分割线**  | `---`                  | 水平分割线              |
    | **图片**   | 拖拽 / 粘贴 / `/image`     | 图片直接插入在文档流中，支持缩放   |
    | **表格**   | `/table`               | 可视化表格编辑，支持增删行列     |
    
    ### 3. 核心交互：Slash Command ( `/` 菜单)
* **触发**：在新行或光标前输入 `/`。
* **展示**：弹出一个浮动搜索菜单。
* **功能**：
  * 支持搜索（输入关键词过滤）。
  * 包含：Text, Heading 1-3, Bullet List, Numbered List, To-do List, Quote, Divider, Code, Image, Table 等。
  * 支持键盘上下键选择，Enter 确认。
    
    ### 4. 导出与存储
* **内部存储**：建议使用 Markdown 纯文本或 JSON (Delta 格式) 存储。
* **导出功能**：
  * 导出为 `.md` (Markdown) 文件。
  * 导出为 `.html` 或 `.pdf` (可选)。

## 三、技术实现要求 (Flutter 特定)

1. **技术栈约束**
   * 基于 **Flutter 3.0+**，健全空安全 (Null Safety)。
   * 推荐基于 `ExtendedTextField` 或 `Quill` 富文本底层逻辑进行深度定制，或参考 `flutter_quill` 但需剥离其双栏逻辑。
   * **禁止**：使用 WebView 套壳。
2. **性能要求**
   * 支持 **10,000 字**以上长文档编辑无明显卡顿。
   * 光标移动响应流畅，无跳位。
3. **架构建议**
   * **State Management**：建议使用 `Provider` 或 `Bloc` 管理文档状态。
   * **数据结构**：建立 `DocumentNode` 模型，区分 `ParagraphNode`, `HeadingNode`, `ListNode` 等。

---

## 四、交付物要求

1. **完整的 Flutter 项目结构**。
2. **核心编辑器 Widget**，可直接嵌入 `Scaffold` 使用。
3. **示例视图**：展示一个包含标题、列表、待办、代码块等常用Markdown语法的完整页面。
4. **注释清晰**：关键逻辑节点必须有代码注释。
